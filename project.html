Project Page

Final prototype + evidence

Our final product is a toy model car mounted on a rotating motor platform. The platform can rotate clockwise and counterclockwise as a controllable output (a scaled-down steering system).

Arduino control Joystick input testing Bidirectional rotation Iterative redesign

Problem statement

Build a driving simulator mechanism that can translate user input into steering motion. We initially considered larger force-feedback mechanisms (like linear actuators), but we prioritized a small, functional prototype that we could iterate on quickly.

Solution overview

Input

A joystick module used for early testing and mapping input to motion. This let us prototype control logic without a full wheel rig.

Controller

Arduino microcontroller to read input signals and generate motor commands.

Output

A rotating motor platform that turns clockwise and counterclockwise. Our toy model car is mounted to make the rotation visible and testable.

Key technical lesson

We learned that if you want a DC motor to spin both directions, you need direction-control hardware—commonly an H-bridge. We tried multiple approaches to make the motor go clockwise and counterclockwise, but without an H-bridge the direction control wasn’t reliable. That realization shaped our final design and documentation.

What we would do next

Integrate an H-bridge and refine the control mapping so the motor direction and speed respond smoothly to input. Then scale from the toy platform to a small steering wheel module with stronger torque.
